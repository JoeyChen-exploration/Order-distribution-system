# 智行调度 — 技术文档

机场接送机智能排单系统，支持订单导入、司机管理、智能派单、手动改派、导出账单。

## 快速启动

```bash
npm install
npm run dev
```

## 排单算法详解（dispatch-engine.ts）

### 第一步：筛选候选司机

同时满足以下条件才进入候选池：

1. **状态在岗**：`status === "available"` 或 `"busy"`（busy 司机可排未来时段）
2. **车型兼容**（见兼容矩阵）
3. **工作时间覆盖**：若司机配置了 `workingHours`，订单 `pickupTime` 必须在其时段内，否则直接排除

**车型兼容矩阵**（司机车型 → 可接订单车型）：

| 司机车型 | 可接订单 |
|---|---|
| 商务型 | 商务型、豪华型、舒适型、经济型 |
| 豪华型 | 豪华型、舒适型、经济型 |
| 舒适型 | 舒适型、经济型 |
| 经济型 | 经济型 |

---

### 第二步：评分

```
总分 = 工作量分 × 0.30
     + 距离分   × 0.35
     - 车型降级罚分
     - 时间冲突罚分 × 0.20
     + 车型完全匹配加分 × 0.15
```

#### 工作量分（Weight 30%）
```
workloadScore = max(0, 100 - 当日已接单数 × 12)
```
当日接单越少分越高，接 8 单以上得 0 分。

#### 距离分（Weight 35%）
```
distanceScore = max(0, 100 - 距离km × 2)
```
起点为"司机有效位置"：
- 当日有已派订单 → 最近一单的**下车点坐标**
- 否则 → 司机的**家庭地址坐标**

终点为订单**上车点坐标**。50 km 以内线性递减，超过 50 km 得 0 分。

> ⚠️ 坐标为 0（未 geocode）时此维度得 0 分，算法退化为只靠工作量和车型评分。

#### 车型降级罚分（直接扣）

| 降级级数 | 扣分 |
|---|---|
| 0（完全匹配） | 0 |
| 降 1 级 | 10 |
| 降 2 级 | 20 |
| 降 3 级 | 30 |

#### 时间冲突判定

```
实际间隔 < 服务类型最小间隔 + 行驶时间估算 → 冲突
```

**服务类型最小间隔**：

| 前单类型 | 后单类型 | 最小间隔 |
|---|---|---|
| 接机/站 | 任意 | 180 分钟 |
| 送机/站 | 接机/站 | 60 分钟 |
| 送机/站 | 送机/站 | 120 分钟 |
| 包车 | 任意 | 120 分钟 |
| 市内约车 | 任意 | 120 分钟 |

**行驶时间估算**：前单下车点 → 后单上车点，按上海城区平均速度 **35 km/h** 计算。

有冲突时 `timeConflictPenalty = 50`，扣 50 × 0.20 = 10 分，但**不直接排除**（有冲突司机仍进推荐列表，只是不作为 bestMatch）。

#### 车型完全匹配加分（Weight 15%）
完全匹配 +15 分，降级接单 +0 分。

---

### 第三步：选最优

- 所有候选按 `totalScore` 降序排列
- `bestMatch` = **分数最高且无时间冲突**的司机
- 若全部有冲突 → 提示"所有候选司机均存在时间冲突，请手动选择"

---

### 批量派单逻辑

```
按时间升序处理（紧急订单优先）
↓
对每条订单运行单条派单算法
↓
确认派单后将该订单以 status=1 加入本地列表
↓
后续订单的冲突检测会感知到同批次已分配的订单
```

## 地理编码

导入时自动调用高德 REST API：

```
GET https://restapi.amap.com/v3/geocode/geo?address=<地址>&key=<KEY>&city=上海
```

- 订单：`pickupAddress` → `pickupLat/pickupLng`，`dropoffAddress` → `dropoffLat/dropoffLng`
- 司机：`homeAddress` → `homeLat/homeLng`
- API Key 位置：`components/order-import-dialog.tsx` 和 `app/api/drivers/import/route.ts`

## 已知限制

| 问题 | 说明 |
|---|---|
| 时间间隔基于取车时间 | 未计算实际服务时长，靠间隔规则内嵌估算，基本合理但不精确 |
| 商务型可降级接豪华型 | MPV 和豪华轿车不可互换，如业务不允许可修改兼容矩阵 |
| 无每日接单上限 | 工作量评分线性降低但无硬上限，司机理论上可被无限派单 |
| 坐标缺失时距离权重失效 | 重新导入数据（geocoding 生效后）可解决 |
| 工作时间为硬过滤 | 超出时段 1 分钟即排除，无缓冲区 |

## 数据导入说明

### 订单（xlsx/csv）

| 字段 | 必填 | 说明 |
|---|---|---|
| 订单号 | ✓ | 唯一标识 |
| 预订车型 | ✓ | 舒适型/豪华型/商务型/经济型 |
| 服务日期 | ✓ | YYYY-MM-DD |
| 上车点 | ✓ | 用于 geocoding |
| 下车点 | ✓ | 用于 geocoding |
| 航班号 | ✓ | |
| 服务类型 | 推荐 | 接机/站、送机/站、包车（影响时间间隔计算） |

### 司机（xlsx/csv）

| 字段 | 必填 | 说明 |
|---|---|---|
| 姓名 | ✓ | |
| 手机号 | ✓ | |
| 车牌号 | ✓ | |
| 车型 | ✓ | 舒适型/豪华型/商务型/经济型 |
| 家庭住址 | 推荐 | 用于 geocoding，影响距离评分 |
| 工作时间 | 推荐 | 格式 HH:MM-HH:MM，支持跨午夜如 22:00-06:00 |

## 账单导出

导出已派单订单（status 1/2/3）为 xlsx，Sheet 名 `礼宾车账单`，共 21 列，与人工账单格式对齐：

订单号、预订车型、服务类型、服务城市、服务日期、三字码、人数、下单时间、航班号、上车点、下车点、车号、司机、司机电话、司机分组、实际车型、架次、公里数、货币、超公里费、夜间服务费
